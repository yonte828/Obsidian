# SysMLによる要求分析と機能分解

[[2025-12-18]]のセミナーメモより、[[MBSE]]（モデルベースシステムズエンジニアリング）における[[SysML]]の活用と要求分析のポイントについてまとめます。

## SysMLとは
Systems Modeling Language。システム工学のための汎用モデリング言語。[[UMLとは|UML]]をベースに拡張されたもの。仕様、分析、設計、検証などの情報を図（ダイアグラム）で表現する。

## 要求分析のポイント
*   **リーダーの役割**: 要求分析における**「粒度」をそろえる**のはリーダーの重要な仕事。メンバーによって記述の細かさがバラバラだと、モデルとして統合できない。
*   **機能分解**: システムを機能単位で分解していく。

## エネルギーフローの絶対定義
機能を定義する際、**エネルギーフロー**に着目する手法が有効。
以下の3要素をセットで定義する。

1.  **IN**: 入力（エネルギー、物質、信号）
2.  **f(x)**: 機能・変換（何をするか）
3.  **OUT**: 出力（変換されたエネルギー、物質、信号）

これを明確にすることで、システム内部の振る舞いを物理現象として矛盾なくモデル化できる。

## モデリングの具体例（ハイブリッド車）
UMLと異なり、ハードウェアや物理法則を含む「システム全体」を記述する例（ハイブリッド車）です。

1.  **要求図 (Requirement Diagram)**
    ```mermaid
    graph TD
        Req1(["燃費を20km/L以上にする"])
        Req2(["衝突安全性を確保する"])
        Comp1(エンジン)
        Comp2(モーター)

        Req1 -- Satisfy --> Comp1
        Req1 -- Satisfy --> Comp2
    ```

2.  **ブロック定義図 (bdd: Block Definition Diagram)**
    ```mermaid
    classDiagram
        class ハイブリッド車{
        }
        class エンジン{
        }
        class モーター{
        }
        class バッテリー{
        }
        class タイヤ{
        }
        class ECU{
        }

        ハイブリッド車 "1" *-- "1" エンジン
        ハイブリッド車 "1" *-- "1" モーター
        ハイブリッド車 "1" *-- "1" バッテリー
        ハイブリッド車 "1" *-- "4" タイヤ
        ハイブリッド車 "1" *-- "1" ECU
    ```

3.  **内部ブロック図 (ibd: Internal Block Diagram)**
    ```mermaid
    graph LR
        Engine[エンジン] --> |トルク| Tire[タイヤ]
        Battery[バッテリー] --> |電力| Motor[モーター]
        Motor --> |トルク| Tire
        ECU[ECU] --> |制御信号| Engine
        ECU --> |制御信号| Motor
    ```

4.  **パラメトリック図 (Parametric Diagram)**
    ```mermaid
    graph TD
        Param1(エンジントルク) --> Eq
        Param2(モータートルク) --> Eq
        Param3(車両重量) --> Eq
        
        Eq{"車両の加速力 = <br>(エンジントルク + モータートルク) <br>/ 車両重量"}
    ```

# UML (Unified Modeling Language) とは

UML（統一モデリング言語）は、主に**ソフトウェア開発**において、システムの構造や振る舞いを可視化するための標準的な図法（言語）です。

## 概要
*   **目的**: 設計図を標準化することで、エンジニア間の認識ズレを防ぎ、コミュニケーションを円滑にする。
*   **特徴**: 「オブジェクト指向」の考え方に基づいて作られている。
*   **関係性**: [[SysMLによる要求分析と機能分解|SysML]]（システムズモデリング言語）の**親**にあたる言語。
    *   **UML**: ソフトウェア中心。
    *   **SysML**: UMLを拡張し、ハード・ソフト・人を含めた「システム全体」に対応させたもの。

## 代表的な図（ダイアグラム）
UMLには多くの図がありますが、大きく「構造」と「振る舞い」に分かれます。

### 1. 構造図（静的な仕組みを表す）
*   **[[クラス図]]**: システムの構成要素（クラス）と、その関係性を表す。最も基本的で重要な図。
*   **コンポーネント図**: ソフトウェアの部品構成や依存関係を表す。
*   **配置図**: ハードウェアとソフトウェアの物理的な配置関係を表す。

### 2. 振る舞い図（動的な動きを表す）
*   **[[シーケンス図]]**: 「時間」の流れに沿って、オブジェクト間のやり取り（メッセージ）を表す。
*   **[[ユースケース図]]**: 「ユーザー（アクター）」がシステムを使って「何ができるか」を表す。要求定義で使われる。
*   **[[ステートマシン図]]**（状態遷移図）: オブジェクトの状態が、イベントによってどう変化するかを表す。組み込み開発で特に重要。
*   **アクティビティ図**: 処理の流れ（フローチャートのようなもの）を表す。

## なぜUMLが必要か
言葉だけの仕様書では、解釈の違い（「ボタンを押したら画面が変わる」の細かい挙動など）が生まれやすい。UMLという「共通の絵」を使うことで、**プログラミングする前の設計図**として機能し、手戻りを防ぐことができる。

## モデリングの具体例（自動販売機）
身近な「自動販売機」を例に、UMLでどう表現するかを示します。

1.  **[[ユースケース図]]（機能要件）**
    ```mermaid
    graph LR
        Actor1("購入者") -- "商品を購入する" --> UC1((商品購入))
        Actor1 -- "釣銭を受け取る" --> UC2((釣銭受取))
        Actor2("補充員") -- "商品を補充する" --> UC3((商品補充))
    ```

2.  **[[クラス図]]（構造設計）**
    ```mermaid
    classDiagram
        class 商品棚{
            - 商品: string
            - 価格: int
            - 在庫数: int
            + 在庫確認()
            + 商品排出()
        }
        class コイン投入口{
            - 投入金額: int
            + お金投入(金額: int)
            + 釣銭排出()
        }
        class 在庫管理部{
            - 総売上: int
            - 在庫情報: map
            + 金額更新(金額: int)
            + 在庫確認(商品: string): int
            + 在庫数更新(商品: string, 数量: int)
        }
        class ディスプレイ{
            - 表示内容: string
            + 金額表示(金額: int)
            + メッセージ表示(メッセージ: string)
        }

        コイン投入口 --> 在庫管理部 : 金額を伝える
        商品棚 --> 在庫管理部 : 在庫数を伝える
        在庫管理部 --> ディスプレイ : 表示内容を更新
    ```

3.  **[[シーケンス図]]（処理の流れ）**
    ```mermaid
    sequenceDiagram
        actor 購入者
        participant コイン投入口
        participant ディスプレイ
        participant 商品ボタン
        participant 商品棚

        購入者->>コイン投入口: お金を入れる
        コイン投入口->>ディスプレイ: 現在金額を表示
        購入者->>商品ボタン: 押す
        商品ボタン->>商品棚: 商品を出す
        商品棚-->>購入者: 商品提供
    ```

4.  **[[ステートマシン図]]（状態遷移）**
    ```mermaid
    stateDiagram-v2
        [*] --> 待機中
        待機中 --> 金額受付中 : お金投入
        金額受付中 --> 販売処理中 : ボタン押下
        販売処理中 --> 待機中 : 商品排出
        待機中 --> [*] : 電源オフ
    ```

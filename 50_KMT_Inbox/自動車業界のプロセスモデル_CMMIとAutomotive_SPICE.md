# 自動車業界のプロセスモデル：CMMIとAutomotive SPICEのわかりやすい解説

[[2025-12-18]]のセミナーメモおよびその後の調査に基づき、専門用語を噛み砕いて解説します。

## 1. CMMI (Capability Maturity Model Integration)
**イメージ：「会社の偏差値（レベル1〜5）」**

*   **概要**: 組織全体の「開発能力」を測る指標です。
*   **特徴**:
    *   **対象**: **会社全体**（または事業部全体）。
    *   **評価**: レベル1〜5の段階評価。「通信簿」のようなもの。
    *   **用途**:
        *   **入札条件**: 「レベル3以上の会社でないと、この大規模システム開発には応募できません」といった足切りに使われる。
        *   **ブランド力**: 「うちはレベル5（最適化レベル）の組織です」とアピールする。
*   **ISO 9001との違い**: ISOは「合格/不合格（免許証）」ですが、CMMIは「レベル（偏差値）」で実力を測ります。

## 2. Automotive SPICE (A-SPICE)
**イメージ：「プロジェクトチームへの抜き打ちチェック」**

*   **概要**: 自動車業界（OEM）がサプライヤーに対して要求する、開発プロセスの品質基準。
*   **特徴**:
    *   **対象**: **特定のプロジェクトチーム**（例：「新型プリウスのブレーキECU開発チーム」）。
    *   **理由**: CMMIで会社全体が優秀でも、**「今回のプロジェクトチームがデスマーチになっていないか？」「ちゃんと手順通り仕事しているか？」** は別問題だからです。
    *   **評価**: 0〜5の能力レベルで評価します。

## 3. Mechanical SPICE (M-SPICE)
**イメージ：「A-SPICEシリーズの『機械設計版』の教科書」**

*   **概要**: 従来のA-SPICEは「ソフトウェア」中心でしたが、それを「機械設計（メカ）」にも適用できるようにした**追加ルールブック（プラグイン）**です。
*   **位置づけ**: A-SPICEもM-SPICEも、どちらも**「モデル（＝理想の仕事リストが書かれた教科書）」**という意味では**同じ土俵**のものです。
    *   **A-SPICE (Software)**: 「コードを書くときはこうしなさい」という教科書。
    *   **M-SPICE (Mechanical)**: 「熱解析や図面作成はこうしなさい」という教科書。
*   **「モデル」とは？**: 数式やプラモデルのことではなく、**「本来やるべきベストプラクティスが書かれたカタログ（チェックリスト）」**のことです。
    *   監査では、自分たちの仕事がこの「教科書（モデル）」通りにできているかをチェックされます。

## まとめ：使い分け
| 名称 | 評価対象 | イメージ |
| :--- | :--- | :--- |
| **CMMI** | **組織全体** | **会社の偏差値**。<br>「この会社は信用できるか？」を見る。 |
| **A-SPICE** | **プロジェクト**<br>(ソフト開発) | **現場のチェック**。<br>「このソフト開発チームは手順を守っているか？」を見る。 |
| **M-SPICE** | **プロジェクト**<br>(メカ開発) | **現場のチェック（メカ版）**。<br>「この機械設計チームは手順を守っているか？」を見る。 |

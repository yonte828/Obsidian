# MS-Projectの活用：日程管理と予実の可視化

[[2025-12-18]]のセミナーメモより、[[MS-Project]]（Microsoft Project）を活用したプロジェクト管理のポイントについてまとめます。

## MS-Projectとは
マイクロソフトのプロジェクト管理ツール。ガントチャート、リソース管理、進捗管理などを統合的に行える。

## 日程の計画と実績の差異（予実）を出す
*   MS-Projectを使う大きなメリットは、**計画（ベースライン）と実績の乖離**を定量的に可視化できること。
*   どのタスクが遅れているのか、その遅れが後工程（クリティカルパス）にどう影響するかを瞬時に把握できる。

## 標準工数の設定：あえて「最大値」を据える
*   **一般的な間違い**: 標準工数を「平均値」や「最短値（ベストケース）」で設定してしまう。
*   **結果**: 必ず突発的な事象で遅れが発生し、達成できなくなる。常に遅れを取り戻す自転車操業になる。
*   **あるべき姿**: 過去の実績工数の**「最大値（ワーストケース）」**を標準として設定する。
*   **効果**:
    *   通常通り進めば時間が余る（工数が余る）。
    *   **余った時間は「仕事の質を高める」「アウトプットの価値を高める」ことに使う。**
    *   これにより、常に余裕を持って高品質な成果物を出す好循環が生まれる。

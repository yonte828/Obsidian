# クラウド環境から社内APIを利用する方法

## はじめに
Excelマクロで使用している[[社内API]]を、[[AI]]が保守しやすい[[クラウド]]環境から利用するための方法を検討。VBAからの移行は、[[AI]]による保守性向上に繋がる。

## 主な接続方法

1.  ### [[APIゲートウェイ]]方式
    *   **概要**: [[社内ネットワーク]]とインターネットの境界に[[APIゲートウェイ]]を設置し、[[クラウド]]上の[[AI]]からのリクエストを安全に社内APIへ中継する。
    *   **メリット**: [[セキュリティ]]が高く、認証やアクセス制御を一元管理できる。社内APIの構造を直接外部に公開せずに済む。
    *   **デメリット**: 構築・運用に専門知識とコストが必要。

2.  ### [[VPN]]接続方式
    *   **概要**: [[クラウド]]環境（AWS, Azure, GCPなど）と[[社内ネットワーク]]を[[VPN]]（Virtual Private Network）で接続する。これにより、[[クラウド]]上の[[AI]]は社内PCと同じネットワークにいるかのようにAPIにアクセスできる。
    *   **メリット**: 多くの[[クラウド]]プラットフォームで提供されており、比較的実現しやすい。
    *   **デメリット**: [[VPN]]接続の常時管理が必要。[[セキュリティ]]設定（ファイアウォールなど）を慎重に行う必要がある。

3.  ### 専用線接続方式
    *   **概要**: [[クラウド]]と[[社内ネットワーク]]を物理的な専用回線で直接接続する（例: AWS Direct Connect）。
    *   **メリット**: 非常に高速・安定・セキュアな通信が保証される。
    *   **デメリット**: 導入コストと月額料金が非常に高額。大規模なシステム向け。

## 次のステップ

これらのどの方法が最適かは、会社の[[セキュリティ]]ポリシーや既存の[[インフラ]]、予算によって大きく異なります。

まずは**社内の情報システム部門やネットワークの管理者に、「クラウド上のシステムから、この社内APIを利用したいのですが、どのような接続方法が可能ですか？」と相談する**ことを強くお勧めします。
